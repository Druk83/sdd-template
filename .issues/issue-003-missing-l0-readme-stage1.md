# Этап [1] Предметная область: не создается README L0 в корне проекта

**Type:** `type:doc`  
**Priority:** `priority:P2`  
**Status:** `status:needs-info`

## Контекст

- **Компонент:** трек разработки документации (этап [1])
- **Область:** структурирование документации проекта
- **Обнаружено:** при анализе процесса формирования базовой документации
- **Impact:** отсутствует входная точка в корне репозитория проекта

## Описание

На этапе [1] Предметная область агент создает первичные артефакты документации в `docs/requirements/`, но не создает README на уровне L0 (в корне проекта — `/README.md`) согласно `.manifest/readmemanifest.md`.

README L0 должен создаваться в конце этапа [1] как финальный шаг и служить главной входной страницей для всего проекта, кратко описывая назначение, возможности и направляя читателя к документации.

## Проблема

- На этапе [1] агент заполняет `docs/requirements/предметная область.md`, но не создает `/README.md` в корне проекта в конце этапа
- Отсутствует файл с базовым описанием проекта, его целей и структуры документации в качестве финального шага
- Нет единой входной точки для понимания что находится в проекте и как его устроена документация
- Нарушается правило в `.manifest/readmemanifest.md` раздел 4.1 (создание L0 README в корне)
- README L0 — критичный артефакт, но упускается как завершающий шаг этапа [1]

## Требования к README L0

Согласно `.manifest/readmemanifest.md` раздел 4.1, README L0 в корне проекта (`/README.md`) должен содержать:

- 1–2 предложения: что за проект и его назначение
- 3–7 bullets: ключевые возможности или характеристики системы
- Краткое описание архитектуры или подхода (high-level)
- Минимальный Quickstart или указание где начать
- Ссылки на основные документы и разделы (особенно на `docs/requirements/`)

## Решение

Требуется доработка трека разработки (`.requirements/трек разработки.md`):
- Добавить в раздел 3.1 "Этап 1: Предметная область" инструкцию по созданию `/README.md` (L0) в корне проекта как **завершающего шага** этапа [1]
- Указать что создание `/README.md` должно быть выполнено после заполнения всех основных артефактов этапа [1]
- Либо создать шаблон для `/README.md` в `.requirements/` и добавить его в трек разработки как последний артефакт этапа [1]

На этапе [1] файл `/README.md` инициализируется в конце с базовой информацией из предметной области.

## Связи

- `.manifest/readmemanifest.md` (стандарты L0/L1/L2)
- `.requirements/трек разработки.md` (этап [1])
- `/README.md` (результат этапа [1] — корень проекта)
