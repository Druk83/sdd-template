# Отсутствует инструмент для рендеринга PlantUML диаграмм в PNG

**Type:** `type:infra`  
**Priority:** `priority:P2`  
**Status:** `status:needs-info`

## Контекст

- **Компонент:** `.tools/` (инструменты для агентов)
- **Область:** генерация визуализаций из *.plantuml файлов
- **Обнаружено:** при анализе производительности агента на примерах архитектур
- **Impact:** агент тратит значительное время на попытки рендеринга диаграмм, либо пропускает этот этап

## Описание

В проекте используется PlantUML для определения диаграмм архитектуры (`.plantuml` файлы в `примеры/Виды архитектур/`). Агент нуждается в инструменте для автоматического рендеринга этих диаграмм в PNG для последующего использования в документации и визуализации.

Инструмента нет в `.tools/`, что заставляет агента либо искать обходные пути, либо пропускать этот этап работы.

## Проблема

- Нет единого инструмента для batch-рендеринга PlantUML в PNG
- Агент не может систематически обновлять визуализации при изменении диаграмм
- Процесс рендеринга требует ручного вмешательства или полагается на внешние сервисы
- Нет интеграции с registry.json и стандартным интерфейсом инструментов

## Требования к инструменту

Инструмент должен (по `.manifest/toolsmanifest.md`):
- Зарегистрирован в `.tools/registry.json`
- Поддерживать `--help` и exit codes (0/1/2)
- Работать в режиме read-only по умолчанию
- Быть кросс-платформенным (Windows/Linux/macOS)
- Иметь README с примерами использования
- Принимать опции для фильтрации файлов и формата вывода

## Решение

Требуется создание нового инструмента в `.tools/plantuml-render/` согласно стандартам манифеста toolsmanifest.md.

## Связи

- `.manifest/toolsmanifest.md` (стандарты)
- `.tools/registry.json` (регистр инструментов)
- `примеры/Виды архитектур/` (примеры PlantUML файлов)
