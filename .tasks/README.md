# .tasks — задачи проекта

Папка содержит машиночитаемые файлы задач, привязанные к коду через PDD (`@todo`).

## Структура

```
.tasks/
  <TASK-ID>.md       — активная задача
  done/
    <TASK-ID>.md     — выполненная задача
  pdd/
    @todoregistry.md — реестр @todo (генерируется инструментом pdd-scan)
```

## Для чего

`.tasks/` — единственное место, где фиксируется **что нужно сделать** и **критерий готовности**.

Связка с кодом работает через PDD: в коде ставится `@todo #<TASK-ID>:...`, в задаче описан контекст и DoD. Реестр всех `@todo` генерируется автоматически инструментом `.tools/pdd/pdd-scan`.

## Жизненный цикл задачи

```
draft → planned → in-progress → done
                             → dropped
                → blocked
```

При переводе в `done` или `dropped` файл перемещается в `.tasks/done/`.

## Идентификаторы

| Тип | Формат | Пример |
|-----|--------|--------|
| Эпик / этап | `A1`, `B2` | `A1.md` |
| Подзадача | `A1-01`, `A1-02` | `A1-01.md` |
| Техдолг | `TD-01` | `TD-01.md` |

TASK-ID уникален в репозитории. Используется в `@todo #<TASK-ID>:<EST>`.
