# ArchiMate — архитектура предприятия

Подход для описания архитектуры в трех слоях: Business Layer (BL), Application Layer (AL), Technology Layer (TL).

## Назначение

ArchiMate используется на этапах [7], [7.1] для:
- Описания бизнес-слоя (акторы, процессы, сервисы)
- Описания прикладного слоя (компоненты, интерфейсы, данные)
- Описания технологического слоя (узлы, артефакты, инфраструктура)
- Создания диаграмм BL.plantuml, AL.plantuml, TL.plantuml

## Глоссарий

### Business Layer (BL)
| Термин | Определение |
|--------|-------------|
| Business Actor | Роль, выполняющая бизнес-поведение |
| Business Role | Ответственность актора |
| Business Process | Последовательность бизнес-действий |
| Business Service | Сервис, предоставляемый бизнесом |
| Business Object | Бизнес-сущность (документ, продукт) |

### Application Layer (AL)
| Термин | Определение |
|--------|-------------|
| Application Component | Модуль ПО |
| Application Service | Сервис, предоставляемый компонентом |
| Application Interface | Точка доступа к сервису |
| Data Object | Структура данных |

### Technology Layer (TL)
| Термин | Определение |
|--------|-------------|
| Node | Вычислительный ресурс (сервер, контейнер) |
| Device | Физическое устройство |
| System Software | ОС, СУБД, middleware |
| Artifact | Развертываемый артефакт |
| Communication Network | Сеть связи |

## Процедура (императив для агента)

### Шаг 1: Business Layer [7 BL]
1. Прочитай результаты этапов [1], [5].
2. Определи бизнес-акторов из use cases.
3. Определи бизнес-процессы из карты процесса.
4. Определи бизнес-сервисы (что система предоставляет).
5. Запиши сущности в `docs/requirements/архитектура/описание BL.md`.
6. Запроси подтверждение: "BL кандидаты: <список>".

### Шаг 2: Application Layer [7 AL]
1. Прочитай результаты этапов [5], [6].
2. Определи компоненты приложения (модули, сервисы).
3. Определи интерфейсы (API, UI).
4. Определи объекты данных (сущности БД).
5. Запиши сущности в `docs/requirements/архитектура/описание AL.md`.
6. Запроси подтверждение: "AL кандидаты: <список>".

### Шаг 3: Technology Layer [7 TL]
1. Прочитай результаты этапа [2] (стек).
2. Определи узлы развертывания (серверы, контейнеры).
3. Определи системное ПО (СУБД, очереди, кэш).
4. Определи сети и коммуникации.
5. Запиши сущности в `docs/requirements/архитектура/описание TL.md`.
6. Запроси подтверждение: "TL кандидаты: <список>".

### Шаг 4: Диаграммы [7.1]
1. Создай `docs/requirements/архитектура/diagrams/BL.plantuml`.
2. Создай `docs/requirements/архитектура/diagrams/AL.plantuml`.
3. Создай `docs/requirements/архитектура/diagrams/TL.plantuml`.
4. Используй ArchiMate-стереотипы PlantUML.
5. Покажи связи между слоями (Realization, Serving, Assignment).

## Критерии готовности

- [ ] Все три слоя (BL, AL, TL) описаны
- [ ] Сущности каждого слоя подтверждены разработчиком
- [ ] Связи между слоями определены
- [ ] Диаграммы созданы и соответствуют описаниям
- [ ] Используются корректные ArchiMate-стереотипы
- [ ] Нет "висящих" сущностей без связей
