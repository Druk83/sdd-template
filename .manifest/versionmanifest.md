# Version Manifest (H2)

Политики и стандарты версионирования шаблонов SDD: единая версия, SemVer, реестр изменений.

---

## version.H3.1 Храни версию в корневом README.md
Единственный источник версии набора шаблонов - `/README.md` (L0).
**Почему важно:**
Множественные источники версий приводят к рассинхронизации, непониманию какая версия актуальна.

---

## version.H4.1 Правило единого источника версии
**Что делать:**
- Указывать версию набора в корневом `/README.md`
- При создании `docs/**` фиксировать версию шаблона в metadata (если применимо)
**Запрещено:**
- Хранить версию в других файлах как primary source
- Дублировать версию в нескольких местах

---

## version.H3.2 Используй единую версию для всех шаблонов
Все шаблоны в `.requirements/` имеют общую версию.
**Почему важно:**
Разные версии шаблонов несовместимы друг с другом, невозможно обеспечить согласованность документации.

---

## version.H4.2 Правило единой версии
При изменении любого шаблона - весь набор получает новую версию.
Нельзя версионировать шаблоны независимо.

---

## version.H3.3 Следуй SemVer
Версионирование по Semantic Versioning обеспечивает понятность изменений.
**Почему важно:**
Без SemVer непонятно какие изменения breaking, какие обратно совместимы.

---

## version.H4.3 Правила SemVer
| Тип версии | Когда использовать |
|------------|-------------------|
| MAJOR (X.0.0) | Несовместимые изменения структуры шаблона |
| MINOR (0.X.0) | Новые секции обратно совместимые |
| PATCH (0.0.X) | Исправления уточнения формулировок |

---

## version.H4.4 Процедура обновления версии
**Что делать при изменении шаблона:**
1. Обновить версию в корневом `/README.md`
2. Обновить дату в реестре шаблонов (`.requirements/трек разработки.md`, раздел 9)
3. Добавить запись в CHANGELOG (если ведется)
4. Проверить совместимость с уже созданными `docs/**` документами

---

## version.H4.5 Веди реестр шаблонов
Реестр дат изменений находится в `.requirements/трек разработки.md`, раздел 9.
**Формат реестра:**
| Шаблон | Последнее изменение |
|--------|---------------------|
| трек разработки.md | YYYY-MM |
| предметная область.md | YYYY-MM |
| ... | ... |

---

## version.H4.6 Не редактируй шаблоны без обновления версии
**Запрещено:**
- Вносить изменения в `.requirements/**` без bump версии
- Использовать разные версии шаблонов в одном проекте
- Пропускать обновление реестра

---

## version.H4.7 Чеклист при обновлении
- [ ] Версия в `/README.md` обновлена
- [ ] Дата в реестре обновлена
- [ ] Тип изменения соответствует SemVer (MAJOR/MINOR/PATCH)
- [ ] Существующие `docs/**` проверены на совместимость
